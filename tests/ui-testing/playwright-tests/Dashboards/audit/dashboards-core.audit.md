# Dashboards-Core E2E Refactor — Audit

Twelve spec files in the `Dashboards-Core` test group were refactored to the
strict zero-tolerance selector policy: Page Object Model only, `data-test`
selectors only (no class names, no element-only locators, no
`getByText` / `getByRole` / `getByLabel` / `text=` / `:has-text` /
`filter({ hasText })` / `data-cy`).

Each spec has its own table below. **Only meaningful changes are listed** —
source `.vue` additions, real bug fixes, and architectural approach shifts.
Routine selector-policy replacements (the bulk of the work, e.g.
`.q-tooltip` → `[data-test="o-tooltip-content"]`, `getByText("X")` →
`pm.someThing.getX()`) are intentionally omitted.

A spec with an empty table contained only routine compliance work — no
behavioural change worth recording.
