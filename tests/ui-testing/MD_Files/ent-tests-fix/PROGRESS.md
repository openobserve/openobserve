## Component Library Patterns (critical тАФ apply to every spec)

These were discovered fixing `logoManagement.spec.js` and apply across all specs:

| Component | Wrong | Correct |
|-----------|-------|---------|
| `<OInput data-test="foo">` | `[data-test="foo"]` (wrapper div) | `[data-test="foo-field"]` (native input) |
| `<OFile data-test="foo">` | `[data-test="foo"]` / `input[data-test="foo"]` | `[data-test="foo-field"]` (hidden input) |
| File input `waitFor` | `{ state: 'visible' }` | `{ state: 'attached' }` |
| Toast assertion | `[data-test="o-toast"]` | `[data-test="o-toast-message"]` |
| Delete confirm dialog OK | any old confirm-btn selector | `[data-test="<dialog-name>"] [data-test="o-dialog-primary-btn"]` |
| OTable row by data value | `filter({ hasText: rowKey })` | intercept GET response тЖТ `findIndex(row => row.id === target)` тЖТ `[data-test="o2-table-row-{index}"]` |
| OTable expand button | `[data-test^="o2-table-expand-"]` scoped to row | `[data-test="o2-table-expand-{rowIndex}"]` (same integer as the row) |

### 3. `jobSearch.spec.js` тАФ 8 tests
**Shard:** Logs-Core
**File:** `tests/ui-testing/playwright-tests/Logs/jobSearch.spec.js`
**Status:** тЬЕ Done (2026-05-29)

**What was fixed (6 issues across 3 files):**
- `logsPage.js selectStreamAndStreamTypeForLogs`: `.fill()` on OSelect wrapper div тЖТ fill popover `input` first; click `[data-test-value="${stream}"]` option instead of stale `stream-toggle` selector
- `logsPageEP.js validateAddJob`: toast text "Job Added Succesfully" тЖТ "Job added successfully" (matches useLogs.ts line 162)
- `logsPageEP.js validateAddJob`: `filter({ hasText: trace_id })` never matched (trace_id not a visible column) тЖТ intercept GET response to find job by id тЖТ rowIndex тЖТ `o2-table-row-{rowIndex}`
- `jobScheduler.js` all methods: same trace_id filter problem тЖТ added `_getJobRowIndex()` helper that intercepts the "Get Jobs" GET response and maps trace_id to positional row index
- `jobScheduler.js restartJobSearch`: restart button disabled until status=2/3 (completed/failed) тЖТ added retry loop matching `exploreJob` pattern
- `jobScheduler.js viewJobDetails`: expand button uses integer rowIndex (`o2-table-expand-{rowIndex}`), not trace_id

**Key pattern discovered:** OTable `row-key` prop is used by Vue internally for list stability but is NOT reflected in any DOM attribute. The `data-test` on rows is always `o2-table-row-{0-based-integer-index}`. To find a specific row by data value: intercept the API GET response, find the item's index in the array, then use that integer index as the locator.
