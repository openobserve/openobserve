feat: add Build tab for visual SQL query builder on logs page

Add a new "Build" tab to the logs page that provides a visual, drag-and-drop
interface for creating SQL queries and visualizations without requiring SQL
knowledge. This feature reuses existing dashboard components to maintain UX
consistency across the application.

## Key Features
- Visual query builder with drag-and-drop field selection
- Real-time SQL query generation with syntax highlighting
- Support for all chart types (bar, line, area, pie, donut, table, metric)
- Full aggregation support (COUNT, SUM, AVG, MIN, MAX)
- WHERE clause filters and GROUP BY breakdowns
- Live chart preview with error/warning display
- Save visualizations directly to dashboards
- "Edit in SQL mode" for manual refinement
- Context synchronization with logs page (stream, time range)

## Implementation Details
- Created BuildQueryTab.vue (~470 lines) - main component
- Created GeneratedQueryDisplay.vue (~220 lines) - SQL display component
- Integrated Build tab into logs page navigation (SearchBar.vue)
- Added event handlers and state management (Index.vue)
- Added CSS styling for Build container (logs-page.scss)
- Added i18n translation key (en.json)
- Reused 6 existing dashboard components (ChartSelection, FieldList,
  DashboardQueryBuilder, PanelSchemaRenderer, ConfigPanel, PanelSidebar)

## Technical Stack
- Vue 3 Composition API with TypeScript
- Quasar Framework for UI components
- useDashboardPanel composable for state management
- Auto SQL generation via makeAutoSQLQuery() function
- Async component loading for performance
- Debounced query generation (500ms)

## Security
- SQL injection prevention via parameterized query builder
- XSS prevention via HTML escaping before syntax highlighting
- Field name validation against stream schema
- Rate limiting through debounced query generation

2. Select a stream
3. Click "Build" tab (4th tab in navigation)
4. Drag fields to create visualizations
